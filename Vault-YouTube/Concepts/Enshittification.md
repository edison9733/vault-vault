# Enshittification

**What it is** - A term coined by writer Cory Doctorow describing the predictable lifecycle of platform decay: platforms initially offer genuine value to attract users and suppliers, then progressively degrade quality to extract profit once users are locked in. Each phase transfers value away from users toward the platform owner.

**Why it matters** - Enshittification names a pattern visible across major platforms — search engines, social networks, app stores, and developer tools. Identifying it helps users and builders recognise lock-in dynamics before exit costs become too high.

**Example** - [[GitHub]] improved significantly after Microsoft's 2018 acquisition (Actions, Codespaces, Copilot). By 2025–2026, declining [[Uptime]], silently lost pull requests, and a [[Remote Code Execution]] vulnerability led high-profile developers including Mitchell Hashimoto to apply the label and migrate away.
