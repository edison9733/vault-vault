# Git

**What it is** - Git is a version control system: software that tracks every change made to a codebase, lets multiple people work on the same files without overwriting each other, and allows rolling back to any previous state. Created by Linus Torvalds in 2005 to manage Linux kernel source code.

**Why it matters** - Virtually all professional software development uses Git. It is the foundation [[GitHub]] is built on; without Git, distributed code collaboration would require manual file merging and constant overwrite conflicts.

**Example** - `git push` sends your local commits to a remote server like GitHub. The 2026 GitHub [[Remote Code Execution]] vulnerability allowed a crafted `git push` to execute arbitrary code on GitHub's own servers.
