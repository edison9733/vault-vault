# CRUD

**What it is** - An acronym for the four fundamental operations on stored data: **Create** (add new records), **Read** (retrieve records), **Update** (modify existing records), **Delete** (remove records). Any complete data management feature implements all four.

**Why it matters** - CRUD maps directly to database operations (INSERT, SELECT, UPDATE, DELETE) and HTTP verbs (POST, GET, PUT/PATCH, DELETE). Building a CRUD app is the canonical exercise for learning any new backend framework or data layer, on-chain or off.

**Example** - A Solana CRUD app stores user entries as on-chain accounts. Create initialises a new account, Read fetches its data, Update modifies fields, and Delete closes the account and reclaims its rent deposit.
